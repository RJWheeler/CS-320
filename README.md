This repository includes artifacts from two major projects completed in CS 320, focusing on software testing, automation, and quality assurance. The Contact Service files demonstrate my ability to design and implement back-end services using Java, while the accompanying JUnit tests highlight my ability to validate functionality and uncover errors through systematic unit testing. The summary and reflections report further illustrates my understanding of testing strategies and how they align with software requirements.

To ensure that my code, program, or software is both functional and secure, I rely on a combination of requirement-driven development and thorough unit testing. Every requirement provided in the project specifications was translated into one or more test cases using JUnit. This allowed me to verify that valid inputs produced correct results and that invalid inputs were properly rejected through exception handling. For example, I tested edge cases such as null values, maximum field lengths, and duplicate IDs to ensure the system behaved reliably under all conditions. By consistently validating both expected and unexpected behaviors, I was able to improve the overall reliability and security of the application.

Interpreting user needs and incorporating them into a program begins with carefully analyzing the requirements and identifying all constraints and expected behaviors. In this project, user needs were expressed through specific rules, such as field length limits, required data, and unique identifiers. I translated these needs directly into code by implementing validation logic in constructors and service methods. I then reinforced this implementation through unit tests that confirmed each requirement was met. This process ensured that the final product accurately reflected the user's expectations and reduced the likelihood of defects.

When approaching software design, I focus on simplicity, modularity, and efficiency. I structured each feature into separate classes, such as Contact, Task, and Appointment, along with their corresponding service classes. This separation of concerns made the code easier to manage, test, and maintain. I also selected efficient data structures, such as HashMaps, to enable fast lookups and updates based on unique identifiers. Additionally, I designed the system with testability in mind, ensuring that each component could be independently verified through unit tests. This approach not only improved code quality but also made debugging and future enhancements more manageable.

Overall, this project strengthened my understanding of how to design, implement, and test software systems that meet defined requirements. It also reinforced the importance of disciplined testing practices and thoughtful design decisions in producing reliable and maintainable code.
